The Problem

Students are required to maintain "Screenshots" of MySQL commands' outputs & Java programs and source code of our programs in Java... There is no way to work at home and at school on a single project. There is just no simple way of taking backups of MySQL Databases.

The process of archiving, storing and transmitting is (currently) achieved in the following steps:

Java/NetBeans:

  • Create the program in NetBeans
  • Finish all the work. Start all over if deleted
  • Create a .zip of the Project folder
  • Email the .zip to yourself OR Forget backups at all!

MySQL:

  • Type the everything in terminal
  • Take a screenshot of the terminal window
  • Paste the screenshot in a MS Word/Google Doc
  • Spend unreasonable amounts of time fixing the layouts and size of the images.
  • Saving the file
  • Email it to yourself

While this method works, it has the following problems:

  • Versioning i.e. keeping track of which .zip is newer or which .zip contains the desired project/program is difficult.
  • Redundancy i.e. it is quite possible that you may have two or more copies of the same program.
  • Keeping track of the date/CW numbers may be difficult.
  • Tedious.
  • Boring.
  • Believe it or not, email's not the best place to store files.
